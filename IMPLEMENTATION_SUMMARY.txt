🗄️ FIRESTORE SCHEMA

Colección: master_food_db
Documento ID: slug (ej: huevo-gallina-grande)

Campos:
  - id: String (slug)
  - name: String (display name)
  - category: String (protein, fat, vegetable, carb)
  - protein_100g: double ✓ PRECISIÓN DOBLE
  - fat_100g: double ✓ PRECISIÓN DOBLE
  - carbs_100g: double ✓ PRECISIÓN DOBLE
  - calories_100g: double ✓ PRECISIÓN DOBLE
  - source: String (internal | external_api)
  - is_verified: boolean
  - created_at: timestamp
  - updated_at: timestamp
  - external_source: String? (openfoodfacts | usda)
  - external_id: String?

🔄 FLUJO DE INTEGRACIÓN

Usuario escribe en MealRegistrationModal
        ↓
FoodSearchAutocomplete triggers búsqueda
        ↓
searchFoodProvider("pollo")
        ↓
FoodRepositoryImpl.getFoodMetadata("pollo")
        ├─ PASO 1: Firestore search en master_food_db
        ├─ PASO 2: Si no → OpenFoodFactsAPI (ready for integration)
        ├─ PASO 3: Map respuesta a FoodModel
        ├─ PASO 4: Guardar en Firestore con source: "external"
        └─ PASO 5: Retornar FoodModel
        ↓
Mostrar FoodSuggestionTile en overlay
        ↓
Usuario selecciona alimento
        ↓
MealLog se crea con foodId y macros pre-rellenados
