  • Complex mapping logic expected 'protein' but received 'Proteinas'
  • If statements never matched, category stayed null
  • Null-check triggered fallback query (WRONG path)

SOLUTION APPLIED:
  ✅ File: food_repository.dart (Lines 59-102)
  ✅ Removed: All if/if/if/if mapping logic
  ✅ Changed: Direct query with parameter as-is
  ✅ Query: .where('metadata.category', isEqualTo: category)
  ✅ No intermediary, no null checks, direct match

BEFORE CODE:
  │ String? plainTextCategory;
  │ if (category == 'protein') plainTextCategory = 'Proteinas';
  │ if (category == 'fat') plainTextCategory = 'Grasas';
  │ if (category == 'vegetable') plainTextCategory = 'Vegetales';
  │ if (category == 'carb') plainTextCategory = 'Carbohidratos';
  │ // Result: plainTextCategory = null
  │ query = query.where('category', isEqualTo: category); // ← WRONG PATH

AFTER CODE:
  │ // Direct query - no mapping
  │ final snapshot = await _firestore

  🎨 BUG #2: RenderFlex Overflow 169 Pixels Right (FIXED ✅)
═══════════════════════════════════════════════════════════════════════════════════

PROBLEM IDENTIFIED:
  ❌ Error: "RenderFlex overflowed by 169 pixels on the right"
  ❌ Location: Food chips display in Nutrition Preferences Step
  ❌ Cause: Food badges don't fit horizontally on narrow screens

ROOT CAUSE:
  • Wrap widget contains multiple food chips
  • Combined width exceeds screen width
  • No horizontal scroll capability
  • Flutter throws RenderFlex error

SOLUTION APPLIED:
  ✅ File: onboarding_screen.dart (Lines 2263-2316)
  ✅ Wrapped: _buildSelectedChips() return in SingleChildScrollView
  ✅ Configured: scrollDirection: Axis.horizontal
  ✅ Child: Wrap widget (for natural text wrapping)

BEFORE CODE:
  │ return Wrap(
  │   spacing: 8,
  │   runSpacing: 8,
  │   children: selectedFoods.map((food) {
  │     // Food chip widgets
  │   }).toList(),
  │ );
  │ // Result: No horizontal scroll, overflow error

AFTER CODE:
  │ return SingleChildScrollView(
  │   scrollDirection: Axis.horizontal,
  │   child: Wrap(
  │     spacing: 8,
  │     runSpacing: 8,
  │     children: selectedFoods.map((food) {
  │       // Food chip widgets
  │     }).toList(),
  │   ),
  │ );
  │ // Result: Horizontal scroll enabled, no overflow ✅

RESULT:
  ✅ No RenderFlex overflow error
  ✅ Chips display with horizontal scroll
  ✅ Mobile-friendly on narrow screens
  ✅ All 105+ food items can be displayed
