### Service Layer Pattern
**Before**: Direct `FirebaseFirestore.instance` calls in UI widgets  
**After**: Dependency-injected services through Riverpod

```
UI Widget
  ↓
Riverpod Provider (ref.read/watch)
  ↓
Service (FoodService, DailyLogService)
  ↓
Repository (Auto-generated by service)
  ↓
Firestore Database
```

### Refactored Files
- `lib/src/features/dashboard/presentation/widgets/fasting_status_widget.dart`
  - Replaced: 3 direct Firestore queries
  - Now uses: `dailyLogService.clearDailyLogForDate()`
